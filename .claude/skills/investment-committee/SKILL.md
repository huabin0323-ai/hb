# AI投资委员会 — 6角色并行股票分析

## 触发条件

以下情况自动启用本skill：
- 用户要求分析某只股票（A股/港股/美股）
- 用户提到"投委会""投资委员会""深度分析""基本面+技术面"
- 用户要求生成"投研报告""风险清单""投资经理总结"

## 核心原则（严格遵守）

1. **仅用于研究展示** — 所有输出标注"不构成任何投资建议"
2. **不输出交易指令** — 禁止出现"买入/卖出/加仓/清仓/增持/减持"等指令性表述
3. **数据可溯源** — 每条数据标注日期和来源，无法确认的标注"待核实"
4. **正反并呈** — 每个角色独立工作，投资经理综合判断，不预设立场

---

## 6角色定义

### 角色一：研究员 Researcher
**职责：** 信息聚合，不做主观判断
**输出：**
1. 最新行情（收盘价、涨跌幅、成交额、总市值、PE/PB）
2. 最新公告（3个月内重要公告摘要）
3. 最新财报核心数据（营收、净利、毛利率、研发费用）
4. 机构研报摘要（评级分布、目标价、核心观点）
5. 行业新闻（行业动态、政策变化、竞争格局）
6. 公司动态（产能、合作、技术突破）

**工具：** WebSearch优先，WebFetch辅助

---

### 角色二：基本面分析师 Fundamental Analyst
**职责：** 财务和竞争分析，不给交易建议
**输出：**
1. 收入结构与变化（分业务收入占比、毛利率趋势、海外占比）
2. 利润质量（近8季度净利趋势、扣非vs归母、经营现金流匹配度）
3. 财务健康度（资产负债率、应收/存货周转、资本开支、自由现金流）
4. 估值分析（PE/PB/PS、历史分位数、可比公司对比、PEG）
5. 行业竞争格局（全球/国内市占率、竞争对手动态、车企自研影响）
6. 核心竞争力评估（技术壁垒、成本优势、客户结构）

**工具：** WebSearch获取财报数据，年报季报为一级来源

---

### 角色三：技术分析师 Technical Analyst
**职责：** 技术面分析，不给交易建议
**输出：**
1. 价格走势概览（日线序列、52周高低点、K线形态）
2. 均线系统（MA5/10/20/60/120/250、多头/空头排列、金叉/死叉）
3. 技术指标（MACD、RSI、KDJ、布林带）
4. 量价分析（成交量趋势、换手率、资金流向）
5. 关键位置（支撑位与压力位，标注强度）
6. 趋势判断（短期/中期/长期，形态分析）

**工具：** WebSearch获取最新股价和技术指标

---

### 角色四：舆情分析师 Sentiment Analyst
**职责：** 市场情绪分析，识别信息差
**输出：**
1. 机构舆情（评级分布、目标价区间、多空逻辑）
2. 媒体舆情（报道基调、热门话题、争议焦点）
3. 社交平台情绪（散户情绪、核心话题、羊群效应）
4. 市场热度指标（融资融券、北向资金、股东户数变化）
5. 争议焦点（看多vs看空逻辑、潜在误读）
6. 事件催化（已发生和即将发生的影响股价事件）

**工具：** WebSearch获取股吧/雪球/媒体内容

---

### 角色五：风险官 Risk Officer
**职责：** 专门寻找反面证据和风险事件
**输出：**
1. 监管与政策风险（国内补贴、欧美关税/法案、反垄断）
2. 行业与竞争风险（产能过剩、车企自研、二线追赶、技术替代）
3. 公司经营风险（客户集中度、原材料波动、海外产能、产品降价）
4. 财务风险（应收坏账、存货减值、汇率、资本开支压力）
5. 资本市场风险（限售解禁、大股东减持、股权质押、融资摊薄）
6. ESG与声誉风险（环保事故、劳工争议、知识产权）
7. 地缘政治风险（技术脱钩、海外工厂地缘、供应链去中国化）

**输出格式：** 每个风险标注【概率：高/中/低】【影响：大/中/小】

**工具：** WebSearch

---

### 角色六：投资经理 Portfolio Manager
**职责：** 不查询新数据，只阅读前5份报告，主持投委会总结
**输出：**
1. 综合评分（基本面/技术面/情绪面/风险面）
2. 核心矛盾识别
3. 情景分析（乐观/中性/悲观，含概率判断）
4. 关键监测指标与观察窗口

**工作方式：** 直接基于前5个agent的回传结果综合

## 执行流程

### Step 1: 接收股票代码
用户提供股票代码或名称（如"300750 宁德时代""600519""Tesla"）

### Step 2: 并行启动5个角色
使用 Agent 工具同时启动5个后台agent：
```
Agent(agentType="general-purpose", run_in_background=true,
      prompt="你是[角色]，分析[股票]...[完整角色prompt]")
```

5个agent并行运行，等待全部完成。

### Step 3: 投资经理综合
基于5份报告回传结果，投资经理角色（主线程）生成：
- 投委会综合讨论
- 多维度评分
- 情景分析

### Step 4: 生成5份交付物

保存到 `output/[日期]/[股票名称]-investment-committee/`

| 文件 | 内容 | 格式 |
|------|------|------|
| `01-总报告.md` | 完整9章综合报告，Word可读 | Markdown |
| `02-数据来源与证据表.csv` | 所有数据的来源与核实状态 | CSV |
| `03-投委会纪要.md` | 6角色会议讨论记录 | Markdown |
| `04-风险清单.md` | 风险矩阵（34项标准模板） | Markdown |
| `05-投资经理总结.md` | 一页纸核心结论+情景分析 | Markdown |

### Step 5: 推送到飞书（可选）
将投委会纪要精简为飞书卡片格式，推送到飞书群。

---

## 输出质量标准

1. **数据完整性** — 每个关键数据点有日期和来源
2. **平衡性** — 看多和看空逻辑都有陈述空间
3. **可操作性** — 情景分析有明确的条件和观察窗口
4. **合规性** — 不输出交易指令，标注免责声明

## 风险清单标准模板

34项风险标准分类：
- 监管与政策（6项）
- 行业与竞争（8项）
- 公司经营（6项）
- 财务（6项）
- 资本市场（5项）
- ESG与声誉（5项）
- 地缘政治（5项）

每项标注：【概率：高/中/低】【影响：大/中/小】，核心风险（高×大）特别标注。

---

## 飞书推送格式

精简为6段卡片：
1. 综合评分表 + 核心矛盾
2. 基本面核心数据
3. 技术面关键位
4. 三大核心风险
5. 情景分析（3情景+概率）
6. 核心观察窗口

使用 lark_md 格式，模板参考 `push_ic_minutes.py`。

## 与 daily-a-share-review 项目的关系

`investment-committee` skill 与 `daily-a-share-review` 项目独立但互补：

## 每日收盘后自动热门股分析

### 定时任务架构

```
15:17 (每个交易日)
  │
  ├─ Windows Task Scheduler: "A股收盘热门股识别"
  │   └─ python hot_stock_daily.py
  │       ├─ 抓取涨停板池 + 龙虎榜 + 要闻
  │       ├─ 综合热度评分（连板×题材×龙虎榜×新闻）
  │       ├─ 保存 output/[日期]/hot_stocks.json
  │       └─ 推送热门股预览卡到飞书
  │
  └─ Claude Cron (session-only, 15:17):
      ├─ 读取 hot_stocks.json
      ├─ 对Top 1-2 启动 /investment-committee 技能
      └─ 推送每只的投委会纪要卡到飞书
```

### 热度评分维度

| 维度 | 权重 | 数据来源 |
|------|:----:|----------|
| 连板高度 | 空间板+50, 5板+35, 3板+20 | 涨停板池 |
| 题材强度 | 主线(≥5家)+20, 支线(≥3家)+10 | 涨停板池 reason 聚合 |
| 早盘秒板 | +10 | 封板时间 < 09:30 |
| 龙虎榜净买 | >5亿+15, >2亿+8 | 龙虎榜明细 |
| 新闻提及 | 每次+3 | 盘后要闻 |

### 手动触发

```bash
# 仅识别热门股 + 推预览
python hot_stock_daily.py

# 指定日期 + 更多股票
python hot_stock_daily.py --date 20260612 --top 5

# 跳过飞书推送
python hot_stock_daily.py --no-push
```

然后对 hot_stocks.json 中的股票执行：
```
/investment-committee [股票代码] [股票名称]
```
